1. How to read this document

Each PBI is one self-contained unit of work that should take a junior developer 1–6 hours. PBIs are numbered in execution order: the lowest-numbered PBI that is still Ready for your role is the next thing you should pick up.

PBI anatomy

ID + Title: stable handle (e.g. PBI-08) you can paste in Slack / commits. Owner: the engineer responsible. Hours: budget + which hour-range of the sprint. Depends on: PBI IDs that must be complete OR mockable before you can finish this one. Files touched: the paths you'll create or modify. Subtasks: ordered checklist to work through. Acceptance: what makes this PBI "done" enough to ship.

Rules of engagement

1. Don't start a PBI before its dependencies, unless you can stub them with sample JSON from backend/ai/samples/. 2. When you start, post PBI-XX started in chat. When done, post PBI-XX done. 3. If you finish early, pull the next PBI for your role. 4. If you're blocked > 30 minutes, escalate to the PM/tech-lead immediately.

PBI-06Storage Adapter (Supabase + Gemini Files Cache)
FS24h · h2–h6

A single Python module that takes an incoming PDF, uploads it to Supabase Storage, uploads the same bytes to Gemini's Files API, and caches the resulting gemini_file_uri on the SourceFile row. Idempotent — calling twice with the same row reuses the cached URI.

Depends on

PBI-03PBI-04

Files touched

  • backend/storage.py
  • backend/tests/test_storage.py

Blocks

PBI-08

Subtasks

  1. Create backend/storage.py.
  2. Initialize Supabase client: supabase = create_client(settings.SUPABASE_URL, settings.SUPABASE_SERVICE_KEY).
  3. Implement upload_pdf(session_id: UUID, file: UploadFile, db: Session) -> SourceFile:
    (a) Generate file_uuid.
    (b) Read bytes from the UploadFile.
    (c) Upload to bucket: supabase.storage.from_("uploads").upload(f"{session_id}/{file_uuid}.pdf", bytes, {"content-type": "application/pdf"}).
    (d) Create SourceFile row with storage_path set, gemini_file_uri=None.
    (e) Commit and return.
  4. Implement ensure_gemini_file(source_file: SourceFile, db: Session) -> str:
    (a) If source_file.gemini_file_uri is set, return it.
    (b) Otherwise: download bytes from Supabase Storage via supabase.storage.from_("uploads").download(storage_path).
    (c) Write to a temp file, then genai_client.files.upload(file=tmp_path).
    (d) Update source_file.gemini_file_uri = uploaded.uri; commit.
    (e) Return the URI.
  5. Implement list_gemini_files(session_id: UUID, db: Session) -> list[str] — for each SourceFile in the session, call ensure_gemini_file and collect URIs.
  6. Write a small pytest in backend/tests/test_storage.py: upload one sample PDF, verify row inserted, call ensure_gemini_file twice, verify URI is reused on the second call (no second Gemini upload).
  7. Add module docstring documenting the three public functions.

Acceptance

  • Calling upload_pdf once writes both to Supabase Storage and DB.
  • Calling ensure_gemini_file twice on the same row only hits Gemini once.
  • Bucket path follows {session_id}/{file_uuid}.pdf convention.

9. Global Definition of Done

These apply to every PBI, on top of its own acceptance criteria.

  1. Code runs. The change builds and the relevant npm run dev / uvicorn starts without new errors.
  2. No secrets in git. .env is gitignored; any new keys are added to .env.example.
  3. Schemas are stable. Don't change BlueprintSchema or PaperSchema fields after they're locked at PBI-07 / PBI-11 — extend, don't rename.
  4. One commit per PBI minimum. Commit message format: PBI-XX: short summary.
  5. Manual smoke test. Before posting "PBI-XX done", click through the relevant flow once on your laptop.
  6. Stub upstream when needed. If your PBI's dependencies aren't ready, mock with sample JSON in backend/ai/samples/ — never wait idle.

Don't try to catch up by working faster. Instead, invoke the next scope cut from build.html §10.2 and inform the team in chat: 1) drop per-section regenerate, 2) drop blueprint review (auto-route to paper), 3) drop autosave (explicit Save button), 4) drop per-question regenerate (text edits only), 5) drop print CSS polish (plain HTML), 6) drop the nudge input.
